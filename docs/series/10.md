## Part A：发表级图表设计

### 为什么默认图表不能直接用

你在前面 9 篇中已经画了很多图——PCA、火山图、热图、气泡图、sashimi plot。它们能清楚地展示结果，但如果直接放到论文或公众号里，会有几个问题：

**字号太小。** ggplot2 默认的 base_size 在电脑屏幕上看着还行，打印出来或者在手机上看就太小了。

**配色不够专业。** 默认配色能区分类别，但和 Nature/Cell 论文中那种精心搭配的色彩方案差距明显。

**元素太多。** 默认图表有灰色背景、网格线、粗边框——这些在发表级图表中通常是多余的。

**不统一。** 不同图之间的字体、字号、配色风格不一致——放在一起看像是不同人做的。

### 定义 Songlab 主题

我们来创建一个统一的 ggplot2 主题，所有图都用它：

```r
# ============================================================
# 文件：analysis/10_figure_refinement.R
# 功能：发表级图表美化 + 全系列结果整合
# ============================================================

library(ggplot2)
library(dplyr)
library(readr)
library(patchwork)     # 多图拼接
library(pheatmap)
library(ggrepel)
library(RColorBrewer)

# ============================================================
# Songlab ggplot2 主题
# 设计原则：
#   - 白色背景，无网格线（干净）
#   - 字号适合打印和手机阅读（base_size = 14）
#   - 坐标轴线条细但清晰
#   - 无多余边框
#   - 统一的字体层级（标题 > 轴标签 > 刻度标签 > 图例）
# ============================================================

theme_songlab <- function(base_size = 14) {
    theme_minimal(base_size = base_size) %+replace%
    theme(
        # 背景
        plot.background = element_rect(fill = "white", color = NA),
        panel.background = element_rect(fill = "white", color = NA),
        panel.grid.major = element_blank(),
        panel.grid.minor = element_blank(),

        # 坐标轴
        axis.line = element_line(color = "grey20", linewidth = 0.4),
        axis.ticks = element_line(color = "grey20", linewidth = 0.3),
        axis.text = element_text(color = "grey30", size = base_size * 0.8),
        axis.title = element_text(color = "grey10", size = base_size,
                                   face = "plain"),

        # 标题
        plot.title = element_text(size = base_size * 1.2, face = "bold",
                                   hjust = 0, margin = margin(b = 8)),
        plot.subtitle = element_text(size = base_size * 0.9,
                                      color = "grey40", hjust = 0,
                                      margin = margin(b = 12)),

        # 图例
        legend.background = element_blank(),
        legend.key = element_blank(),
        legend.text = element_text(size = base_size * 0.8),
        legend.title = element_text(size = base_size * 0.9),

        # 边距
        plot.margin = margin(t = 10, r = 10, b = 10, l = 10)
    )
}

# Songlab 配色方案
songlab_colors <- list(
    # 双组对比（Control vs Treatment）
    two_group = c("#4A90D9", "#D94A4A"),

    # 差异表达方向
    volcano = c(Up = "#C0392B", Down = "#2980B9", NS = "grey75"),

    # 多分类（最多 8 色）
    categorical = c("#E64B35", "#4DBBD5", "#00A087", "#3C5488",
                    "#F39B7F", "#8491B4", "#91D1C2", "#DC9C26"),

    # 连续色阶（热图）
    heatmap_diverge = colorRampPalette(c("#2166AC", "#F7F7F7", "#B2182B"))(100),
    heatmap_sequential = colorRampPalette(c("#F7FBFF", "#2171B5"))(100)
)

cat("Songlab 主题和配色方案定义完成\n")
```

💡 **为什么自定义主题而不是直接用 theme_classic()**

> `theme_classic()` 是一个不错的起点，但它的默认字号偏小、坐标轴线条偏粗、没有处理标题和副标题的层级关系。自定义主题的价值在于**统一性**——你的论文中所有图看起来都像同一个人做的，而且每一个视觉元素都是有意识的设计决策，而不是"默认就这样"。

    design = ~ Group

### 重做火山图

```r
# ============================================================
# 重做火山图（发表级版本）
# ============================================================

res_df <- read_tsv("results/deseq2_all_genes.tsv")

res_plot <- res_df %>%
    filter(!is.na(padj)) %>%
    mutate(
        Sig = case_when(
            padj < 0.05 & log2FoldChange > 1  ~ "Up",
            padj < 0.05 & log2FoldChange < -1 ~ "Down",
            TRUE ~ "NS"
        ),
        # 限制 y 轴范围，避免极端值把图拉变形
        neg_log10_padj = pmin(-log10(padj), 50)
    )

# 标记关键基因
key_genes <- c("AT5G52310", "AT5G05410", "AT4G34000",
               "AT1G20440", "AT5G52300")
gene_labels <- c("RD29A", "DREB2A", "ABF3", "COR47", "RD29B")
names(gene_labels) <- key_genes

res_plot$Label <- gene_labels[res_plot$Gene]

# 统计数字（显示在图上）
n_up <- sum(res_plot$Sig == "Up")
n_down <- sum(res_plot$Sig == "Down")

p_volcano_pub <- ggplot(res_plot,
                         aes(x = log2FoldChange,
                             y = neg_log10_padj,
                             color = Sig)) +
    geom_point(alpha = 0.4, size = 0.8) +
    geom_point(data = filter(res_plot, !is.na(Label)),
               size = 2.5, alpha = 1) +
    geom_text_repel(aes(label = Label), size = 3.5,
                     color = "grey10",
                     fontface = "italic",
                     max.overlaps = 20,
                     segment.color = "grey60",
                     segment.size = 0.3) +
    scale_color_manual(values = songlab_colors$volcano) +
    geom_hline(yintercept = -log10(0.05),
               linetype = "dashed", color = "grey60",
               linewidth = 0.3) +
    geom_vline(xintercept = c(-1, 1),
               linetype = "dashed", color = "grey60",
               linewidth = 0.3) +
    annotate("text", x = 5, y = 48,
             label = paste0("Up: ", n_up),
             color = "#C0392B", size = 4, fontface = "bold") +
    annotate("text", x = -5, y = 48,
             label = paste0("Down: ", n_down),
             color = "#2980B9", size = 4, fontface = "bold") +
    labs(
        x = expression(log[2]~"Fold Change"),
        y = expression(-log[10]~"adjusted p-value")
    ) +
    xlim(c(-8, 8)) +
    theme_songlab() +
    theme(legend.position = "none")

ggsave("results/figures/pub_volcano.png", p_volcano_pub,
       width = 7, height = 6, dpi = 300)
ggsave("results/figures/pub_volcano.pdf", p_volcano_pub,
       width = 7, height = 6)
```

⚠️ **踩坑预警：投稿时 PNG 还是 PDF？**

> 不同期刊对图片格式要求不同。一般规则：
>
> - **投稿初稿**：PNG（300 DPI）足够，文件小，上传方便
> - **修回/终稿**：PDF 或 TIFF（600 DPI），矢量格式无损放大
> - **公众号**：PNG（300 DPI，宽度 1080px）
>
> 我们每张图都同时导出 PNG 和 PDF。`ggsave()` 的输出格式由文件后缀自动决定——`.png` 导出位图，`.pdf` 导出矢量。

## Part B：全系列结果整合

### 从 9 篇分析中提炼科学故事

分散在 9 篇文章中的结果需要被整合成一个连贯的叙事。我们来做一个"summary figure"——一张图讲完整个故事。

```r
# ============================================================
# 整合图：用 patchwork 拼接多个子图
# 布局：
#   A: PCA（实验有效性）
#   B: 火山图（差异表达全景）
#   C: GO 富集条形图（功能分类）
#   D: TF 活性条形图（调控核心）
# ============================================================

# 子图 A：PCA（已经画好了）
pA <- p_pca_pub +
    labs(title = "A") +
    theme(plot.title = element_text(size = 18, face = "bold"))

# 子图 B：火山图（已经画好了）
pB <- p_volcano_pub +
    labs(title = "B") +
    theme(plot.title = element_text(size = 18, face = "bold"))

# 子图 C：GO 富集 top 8（上调基因）
go_data <- read_tsv("results/go_bp_up_ORA.tsv") %>%
    head(8) %>%
    mutate(Description = factor(Description,
                                 levels = rev(Description)))

pC <- ggplot(go_data, aes(x = -log10(p.adjust), y = Description)) +
    geom_col(fill = "#C0392B", alpha = 0.8, width = 0.7) +
    labs(x = expression(-log[10]~"adj. p-value"),
         y = "", title = "C") +
    theme_songlab() +
    theme(
        plot.title = element_text(size = 18, face = "bold"),
        axis.text.y = element_text(size = 10)
    )

# 子图 D：TF 活性
tf_data <- read_tsv("results/decoupler_tf_activity.tsv") %>%
    mutate(
        TF_name = case_when(
            source == "AT5G05410" ~ "DREB2A",
            source == "AT4G34000" ~ "ABF3",
            source == "AT2G47190" ~ "MYB2",
            source == "AT1G01720" ~ "ANAC002",
            source == "AT5G57050" ~ "ABI2",
            TRUE ~ source
        ),
        Direction = ifelse(score > 0, "Activated", "Repressed")
    )

pD <- ggplot(tf_data, aes(x = reorder(TF_name, score),
                            y = score, fill = Direction)) +
    geom_col(width = 0.7, alpha = 0.9) +
    coord_flip() +
    scale_fill_manual(values = c("Activated" = "#C0392B",
                                  "Repressed" = "#2980B9")) +
    geom_hline(yintercept = 0, color = "grey30", linewidth = 0.3) +
    labs(x = "", y = "TF Activity Score", title = "D", fill = "") +
    theme_songlab() +
    theme(
        plot.title = element_text(size = 18, face = "bold"),
        legend.position = c(0.8, 0.2)
    )

# 拼接
p_combined <- (pA | pB) / (pC | pD) +
    plot_annotation(
        title = "干旱胁迫下拟南芥转录组的多层次分析",
        theme = theme(
            plot.title = element_text(size = 16, face = "bold",
                                       hjust = 0.5)
        )
    )

ggsave("results/figures/pub_combined_figure.png", p_combined,
       width = 16, height = 14, dpi = 300)
ggsave("results/figures/pub_combined_figure.pdf", p_combined,
       width = 16, height = 14)

cat("整合图已保存\n")
```

### 一键复现脚本

⚠️ **踩坑预警：`set -euo pipefail` 的含义**

> 这行放在脚本开头，意思是：
>
> - `-e`：任何命令返回非零退出码（报错）时立即停止，不继续执行后面的命令
> - `-u`：使用未定义的变量时报错（避免拼写错误导致变量为空）
> - `-o pipefail`：管道中任何一个命令失败时整个管道算失败
>
> 没有这行的话，一个步骤报错了脚本还会继续跑，你可能直到最后才发现中间某步出了问题——但那时候所有后续结果都已经不可信了。**所有 bash 脚本的第一行都应该加上这个。**
