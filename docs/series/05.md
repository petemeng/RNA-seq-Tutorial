前面四篇，我们把原始测序数据一路处理到了 counts matrix——每个基因在每个样本中有多少条 reads。这一篇，我们终于要回答那个最核心的问题：**干旱处理后，哪些基因的表达发生了显著变化？**

第 6 点是这篇的核心。拿到一张 DEG 列表不难，难的是知道这张列表值不值得信。

## 为什么需要统计检验，而不是直接比大小

上一篇结尾，我们肉眼看了一下 counts matrix，发现基因 AT1G01010 在 Control 组约 400、Drought 组约 850——翻了一倍。这算"显著变化"吗？

答案是：**不知道。光看数字大小不够。**

想象一个极端的例子：某个基因在 Control 的三个重复中分别是 100、500、900（均值 500），在 Drought 中分别是 600、700、800（均值 700）。均值确实从 500 涨到了 700，但 Control 组内部的波动太大了——100 到 900，跨了 9 倍。在这种情况下，700 相对于 500 的变化，可能只是随机波动，而不是真正的生物学效应。

**差异分析的本质是：在考虑了组内变异的情况下，判断组间差异是否大到不太可能由随机波动解释。**

DESeq2 是目前 RNA-seq 差异分析中使用最广泛的工具之一。它背后的统计模型是负二项分布（negative binomial distribution）——你不需要理解数学细节，只需要知道它做了两件关键的事：一是估计每个基因的组内变异程度，二是在此基础上检验组间差异是否显著。

## 用 tximport 导入 Salmon 结果

DESeq2 需要的输入是基因水平的 counts。但 Salmon 输出的是转录本水平的定量（quant.sf）。`tximport` 负责把转录本水平的数据汇总到基因水平——它不是简单地把同一个基因的转录本 counts 加起来，而是用一种更精确的方式校正了转录本长度的影响。

我们把整个差异分析写成一个 R 脚本：

```r

# 文件：analysis/05_differential_expression.R
# 功能：从 Salmon 定量结果到差异表达基因列表
# 输入：results/salmon/*/quant.sf + data/sample_info.tsv

library(DESeq2)
```

## PCA：先看看样本的全局关系

在跑差异分析之前，我们先做一个 PCA（主成分分析），看看 6 个样本在全局表达模式上是什么关系。

PCA 的直觉理解：它把每个样本看作一个在 27,655 维空间中的点（每个基因是一个维度），然后找到能最大程度区分样本的两个方向（PC1 和 PC2），把这些高维点投影到一张二维图上。

**如果你的实验设计是有效的，你应该在 PCA 图上看到：同一组的样本聚在一起，不同组的样本分开。**

```r
# ============================================================
# Step 4: 构建 DESeqDataSet 并做 PCA
# ============================================================

# 创建 DESeqDataSet 对象
# design = ~ Group: 告诉 DESeq2 我们要比较 Control vs Drought
dds <- DESeqDataSetFromTximport(
    txi,
    colData = data.frame(
        Group = factor(sample_info$Group,
                       levels = c("Control", "Drought"))
    ),
    design = ~ Group
)

cat("DESeqDataSet 创建完成\n")
cat("  基因数:", nrow(dds), "\n")
cat("  样本数:", ncol(dds), "\n")
```

```
📊 输出：
DESeqDataSet 创建完成
  基因数: 27655
  样本数: 6
```

```r
# ============================================================
# Step 4b: 方差稳定化变换 + PCA
# vst(): 对 counts 做方差稳定化变换（variance stabilizing
#         transformation），让低表达和高表达基因的变异度
#         处于同一尺度——PCA 需要这个，否则高表达基因会
#         主导所有结果
# ============================================================

vsd <- vst(dds, blind = TRUE)

# 用 DESeq2 内置函数画 PCA
pca_data <- plotPCA(vsd, intgroup = "Group", returnData = TRUE)
pca_var <- round(100 * attr(pca_data, "percentVar"))

p_pca <- ggplot(pca_data, aes(x = PC1, y = PC2,
                               color = Group, label = name)) +
    geom_point(size = 4) +
    geom_text_repel(size = 3, max.overlaps = 10) +
    scale_color_manual(values = c("Control" = "#4A90D9",
                                   "Drought" = "#D94A4A")) +
    labs(
        title = "PCA：样本全局表达模式",
        x = paste0("PC1 (", pca_var[1], "% variance)"),
        y = paste0("PC2 (", pca_var[2], "% variance)")
    ) +
    theme_minimal(base_size = 14) +
    theme(legend.position = "right")

ggsave("results/figures/pca_plot.png", p_pca,
       width = 8, height = 6, dpi = 300)

cat("PCA 图已保存\n")
```

<!-- 图 1 位置：PCA 图 -->
![](../assets/rnaseq/pca_plot-fa1d645eef.png)

**图 1：PCA 显示 Control 和 Drought 样本在全局表达模式上清晰分离。** PC1 解释了约 78% 的方差，主要反映了干旱处理效应——Control 的 3 个重复聚在左侧，Drought 的 3 个重复聚在右侧。PC2 解释了约 9% 的方差，反映了组内的生物学变异。同一组的三个重复紧密聚集，说明实验的重复性良好。

**这张图告诉我们三件事：**

**实验有效。** 处理组和对照组明显分开，说明干旱处理确实导致了基因表达的大规模变化。如果两组完全混在一起，后面做差异分析就没有意义了。

**没有异常样本。** 六个点各自归属自己的组，没有哪个点跑到了对面。如果有一个 Drought 样本混在 Control 里面，你就需要回去检查这个样本是不是标注错了或者处理有问题。

**重复性好。** 同一组的三个点紧密聚在一起。如果组内离散度很大（三个 Control 样本散得到处都是），差异分析的统计效力就会很低。

⚠️ **踩坑预警：PCA 上两组没分开怎么办**

> 如果你做出来的 PCA 图上两组完全没有分离，先不要慌。按顺序检查：
>
> 1. **样本标注是不是搞错了？** 检查 sample_info.tsv 里的分组信息是否和实际实验对应。
> 2. **有没有 batch effect？** 如果你的样本是不同批次测序的，PCA 上可能按批次分开而不是按处理分开。这时候需要在 design formula 中加入批次因素。
> 3. **处理效应真的很弱？** 有些实验条件下，基因表达的变化确实很小。这不意味着分析失败，但你需要接受 DEG 数量可能很少的结果。

## 运行 DESeq2 差异分析

```r
# ============================================================
# Step 5: 运行差异分析
# DESeq() 内部完成三件事：
#   1. 估计 size factors（校正测序深度差异）
#   2. 估计每个基因的离散度（dispersion）
#   3. 对每个基因做负二项分布的 Wald 检验
# ============================================================

dds <- DESeq(dds)

# 提取差异分析结果
# contrast: 比较 Drought vs Control
# 也就是说，log2FoldChange > 0 表示基因在 Drought 中上调
res <- results(dds,
               contrast = c("Group", "Drought", "Control"),
               alpha = 0.05)

cat("差异分析结果摘要：\n")
summary(res)
```

```
📊 输出：
out of 27655 with nonzero total read count
adjusted p-value < 0.05
LFC > 0 (up in Drought)  : 3245, 12%
LFC < 0 (down in Drought): 2876, 10%
outliers [1]              : 23, 0.083%
low counts [2]            : 8765, 32%
(mean count < 5)
[1] see 'cooksCutoff' argument
[2] see 'independentFiltering' argument
```

**3,245 个基因在干旱处理后显著上调，2,876 个显著下调。** 一共约 6,100 个差异表达基因，占有效基因的 22% 左右。对于干旱胁迫这样的强处理来说，这个比例是合理的。

输出中另外几个数字也值得关注：

**outliers: 23** —— DESeq2 会自动检测 counts 分布异常的基因（某个样本的 counts 远远偏离其他样本），并将其 p 值设为 NA。23 个 outlier 非常少，说明数据质量好。

**low counts: 8,765 (32%)** —— 有近三分之一的基因因为表达量太低（均值 < 5 counts）而被独立过滤（independent filtering）自动排除。这是 DESeq2 的一个聪明设计：低表达基因的统计检验不可靠，把它们排除可以减少多重检验校正的负担，从而提高真正差异基因的检出力。

## 关键一步：用 p 值直方图判断结果是否可信

**这是整篇文章最重要的一节。** 大多数教程在跑完 DESeq2 后直接开始画火山图。但在那之前，有一个检查必须先做——它能告诉你整个差异分析的结果是否值得信任。

```r
# ============================================================
# Step 6: p 值直方图——差异分析的"体检报告"
# 看 raw p-value（不是 adjusted p-value）的分布
# ============================================================

# 提取非 NA 的 raw p-values
pvals <- res$pvalue[!is.na(res$pvalue)]

p_hist <- ggplot(data.frame(pvalue = pvals), aes(x = pvalue)) +
    geom_histogram(bins = 50, fill = "#4A90D9",
                   color = "white", alpha = 0.8) +
    labs(
        title = "Raw p-value 分布",
        subtitle = "左侧富集 = 真实信号 | 均匀 = 无差异 | 右侧富集 = 模型问题",
        x = "p-value",
        y = "基因数"
    ) +
    theme_minimal(base_size = 14)

ggsave("results/figures/pvalue_histogram.png", p_hist,
       width = 8, height = 5, dpi = 300)
```

<!-- 图 2 位置：p 值直方图 -->
![](../assets/rnaseq/pvalue_histogram-d606584fb5.png)

**图 2：raw p-value 的直方图。** 左侧（p ≈ 0 附近）有一个明显的高峰，其余部分大致均匀分布。

**怎么读这张图——这可能是你在生信分析中学到的最有用的一个技巧：**

p 值直方图的形状直接反映了你的差异分析是否合理。它只有三种基本形态：

**形态 A：左侧富集（靠近 0 的峰 + 平坦的尾部）。** 这是我们期望看到的健康形态。靠近 0 的峰代表真正差异表达的基因——它们的 p 值很小。平坦的尾部代表没有差异的基因——在零假设下，p 值应该服从均匀分布。我们的数据就是这种形态。

**形态 B：完全均匀分布，没有左侧峰。** 意味着你的数据中没有（或几乎没有）真正差异的基因。这可能是因为处理效应很弱，也可能是你的分组搞错了。

**形态 C：右侧富集（靠近 1 的峰）或"U 形"。** 这意味着你的统计模型有问题——可能是 p 值被保守地膨胀了。常见原因包括：没有校正 batch effect、样本标注错误、design formula 写错了、或者样本量太小。如果你看到这种形态，**不要继续分析**，先排查原因。

💡 **为什么大多数教程不教 p 值直方图？**

> 因为大多数教程关注的是"怎么跑出结果"，而不是"怎么判断结果对不对"。但在实际研究中，判断结果的可信度比得到结果本身更重要。
>
> p 值直方图是最简单、最快速的质量检查。养成习惯：每次跑完差异分析，第一件事不是画火山图，而是画 p 值直方图。

## 结果可视化：火山图

p 值直方图确认了结果可信，现在我们可以放心地可视化了。

```r
# ============================================================
# Step 7: 火山图（Volcano Plot）
# 横轴：log2 fold change（变化幅度）
# 纵轴：-log10(adjusted p-value)（统计显著性）
# 阈值：|log2FC| > 1 且 padj < 0.05
# ============================================================

res_df <- as.data.frame(res) %>%
    mutate(
        Gene = rownames(res),
        Sig = case_when(
            padj < 0.05 & log2FoldChange > 1  ~ "Up",
            padj < 0.05 & log2FoldChange < -1 ~ "Down",
            TRUE ~ "NS"
        )
    )

# 标记关键的干旱响应基因（用于后续验证）
drought_markers <- c("AT5G52310",  # RD29A
                     "AT5G05410",  # DREB2A
                     "AT2G21490",  # LEA 家族
                     "AT1G20440",  # COR47
                     "AT5G52300")  # RD29B

res_df$Label <- ifelse(res_df$Gene %in% drought_markers,
                        res_df$Gene, NA)

# 统计
cat("差异基因统计（|log2FC|>1, padj<0.05）：\n")
cat("  上调:", sum(res_df$Sig == "Up", na.rm = TRUE), "\n")
cat("  下调:", sum(res_df$Sig == "Down", na.rm = TRUE), "\n")
cat("  不显著:", sum(res_df$Sig == "NS", na.rm = TRUE), "\n")
```

```
📊 输出：
差异基因统计（|log2FC|>1, padj<0.05）：
  上调: 1876
  下调: 1543
  不显著: 24236
```

```r
p_volcano <- ggplot(res_df, aes(x = log2FoldChange,
                                 y = -log10(padj),
                                 color = Sig)) +
    geom_point(alpha = 0.5, size = 1) +
    geom_text_repel(aes(label = Label), size = 3,
    color = "black", max.overlaps = 20) +
    scale_color_manual(
        values = c("Up" = "#D94A4A",
                   "Down" = "#4A90D9",
                   "NS" = "grey70"),
```

## 📖 与原文比较

我们用的DRP010324 对应研究的原始论文报告了约 3,200 个差异表达基因（padj < 0.05, |log2FC| > 1）。我们得到了 3,419 个——略多一些，但在同一数量级。

差异可能来自两个地方：第一，定量方法不同——原文使用 STAR + featureCounts，我们使用 Salmon + tximport。如上一篇讨论的，Salmon 对多比对 reads 的处理方式不同，可能导致部分低表达基因的 counts 有细微差异。第二，基因注释版本可能不同——不同版本的 GTF 包含的基因数量略有差异。

关键是方向一致：已知的干旱响应基因（RD29A、RD29B、DREB2A、COR47、LEA 家族）在我们的结果中全部显著上调，和几十年来的文献完全吻合。

**这种"和原文结果方向一致、数量接近但不完全相同"是最正常的结果。** 如果你的 DEG 数量和原文差了一个数量级（比如原文报 3000 个你只找到 100 个，或者你找到了 15000 个），那就需要仔细排查原因。

## 本篇小结

这一篇我们完成了 RNA-seq 分析的最核心任务——差异表达分析。

从 Salmon 定量结果出发，用 tximport 导入 DESeq2，做了 PCA 确认实验有效，运行差异分析得到 3,419 个显著差异基因，用火山图和热图展示了结果，最后和原始论文做了比较。

但这一篇最重要的内容不是上面这些步骤，而是中间那个 **p 值直方图**。它是你判断差异分析结果是否可信的第一道防线。记住：

- 左侧富集 + 平坦尾部 → 结果可信
- 完全均匀 → 无差异（或分组错误）
- 右侧富集或 U 形 → 模型有问题，不要继续

养成习惯，每次差异分析后先看这张图。
