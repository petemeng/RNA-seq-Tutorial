## 本篇目标

前面两篇（第 5-6 篇）我们做了差异分析和通路富集，找到了干旱处理后变化的基因，以及它们参与的生物学过程。但这些分析有一个共同的局限：**它们把每个基因当作独立个体来看待。**

现实中，基因不是孤立工作的。几十个甚至几百个基因经常协同表达——它们可能被同一个转录因子调控，可能参与同一个代谢通路，可能共同应对同一种环境信号。差异分析看的是"哪个基因变了"，WGCNA 看的是**"哪些基因总是一起变"**。

读完这一篇，你会：

1. 理解共表达网络的直觉含义——不需要图论背景
2. 用 WGCNA 识别出共表达基因模块
3. 找到与干旱胁迫最相关的模块
4. 从关键模块中提取 hub gene（核心基因）
5. 理解 WGCNA 的结果能告诉你什么，以及它**不能**告诉你什么

## WGCNA 在做什么：一个直觉解释

假设你在观察一个班级里 30 个学生一周的行为。你发现有一组学生（小明、小红、小华）的行为模式高度同步——小明今天活跃，小红和小华也活跃；小明明天安静了，那两个也安静了。另一组（大壮、大力、大勇）也是这样——他们之间总是同步的，但和小明那一组的节奏不同。

WGCNA 做的就是这件事，只不过"学生"变成了基因，"行为"变成了表达量，"一周的观察"变成了多个样本。

具体来说，WGCNA 会：

1. 计算所有基因两两之间的表达相关性
2. 根据相关性构建一个加权网络
3. 用层级聚类把高度相关的基因分成若干"模块"（module）
4. 检查每个模块和你关心的表型（比如"是否干旱处理"）之间的关联

最终你会得到这样的结论："某个共表达模块和干旱处理高度相关——这个模块很可能代表了干旱防御的核心调控网络。"

## 一个重要的前提：样本量

**WGCNA 对样本量有要求。** 官方推荐至少 15 个样本。我们只有 6 个样本——这意味着我们的 WGCNA 结果在统计上不如有 30 个样本那么稳健。

为什么还要做？两个原因。第一，6 个样本仍然能揭示最强烈的共表达模式——只是微弱的模式可能会被遗漏。第二，我们做这个教程的目的是教方法，当你把这套流程用到自己的数据上（通常有更多样本）时，结果会更可靠。

💡 **如果你的样本数 < 15**

> WGCNA 仍然可以跑，但需要对结果保持更多怀疑。具体来说：
>
> - 降低你对模块数量的期望——样本少的时候，WGCNA 倾向于把基因分成少数几个大模块
> - 不要过度解读弱相关的模块——只关注那些模块-表型相关性非常强（|r| > 0.8）的模块
> - 用交叉验证的思路：WGCNA 发现的 hub gene 最好能用差异分析（第 5 篇）的结果来佐证

## 数据准备

WGCNA 的输入是归一化后的表达矩阵——和差异分析的输入一样。但我们需要做一些额外的过滤：只保留表达量足够高且变异足够大的基因。低表达基因和几乎不变化的基因只会给网络引入噪声。

```r
# ============================================================
# 文件：analysis/07_wgcna.R
# 功能：加权基因共表达网络分析
# 输入：Salmon 定量结果（通过 tximport + DESeq2 归一化）
# 输出：基因模块、模块-表型关联、hub genes
# ============================================================

library(WGCNA)
library(DESeq2)
library(tximport)
library(ggplot2)
library(dplyr)
library(readr)
library(pheatmap)

# WGCNA 使用多线程
allowWGCNAThreads(4)
```

```r
# ============================================================
# Step 1: 加载数据和归一化
# 复用第 5 篇的 tximport 和 DESeq2 流程
# ============================================================

sample_info <- read_tsv("data/sample_info.tsv")
sample_info$files <- file.path("results", "salmon",
                                sample_info$Run, "quant.sf")

tx_names <- read_tsv(sample_info$files[1])$Name
tx2gene <- data.frame(
    TXNAME = tx_names,
    GENEID = sub("\\.\\d+$", "", tx_names)
)

txi <- tximport(sample_info$files, type = "salmon",
                tx2gene = tx2gene, txOut = FALSE)

dds <- DESeqDataSetFromTximport(
    txi,
    colData = data.frame(
        Group = factor(sample_info$Group,
                       levels = c("Control", "Drought"))
    ),
    design = ~ Group
)

# 方差稳定化变换（WGCNA 推荐用 vst 或 log2(CPM+1)）
vsd <- vst(dds, blind = TRUE)
mat <- assay(vsd)

cat("原始矩阵维度:", nrow(mat), "基因 ×", ncol(mat), "样本\n")
```

```
📊 输出：
原始矩阵维度: 27588 基因 × 6 样本
```

```r
# ============================================================
# Step 2: 基因过滤
# 只保留"有足够表达且有足够变异"的基因
# 这两个条件缺一不可：
#   - 低表达基因的相关性估计不稳定
#   - 不变化的基因对网络没有信息贡献
# ============================================================

# 过滤标准 1：平均表达量 > 10 counts
dds_counts <- counts(dds, normalized = TRUE)
keep_expr <- rowMeans(dds_counts) > 10

# 过滤标准 2：变异系数（CV）排名前 5000 的基因
# 为什么限制在 5000？因为 WGCNA 的计算量和基因数的平方成正比
# 5000 个基因已经足够捕捉主要的共表达模式
gene_var <- apply(mat, 1, var)
top_var <- names(sort(gene_var[keep_expr], decreasing = TRUE))[1:5000]

# 提取过滤后的表达矩阵
# 注意 WGCNA 要求矩阵的行是样本、列是基因（和常规相反）
datExpr <- t(mat[top_var, ])

cat("过滤后矩阵维度:", nrow(datExpr), "样本 ×",
    ncol(datExpr), "基因\n")
```

```
📊 输出：
过滤后矩阵维度: 6 样本 × 5000 基因
```

⚠️ **踩坑预警：矩阵方向搞反了**

> WGCNA 要求输入矩阵的**行是样本、列是基因**。这和几乎所有其他生信工具的习惯（行是基因、列是样本）相反。如果你忘了转置，WGCNA 会尝试计算 6 × 6 的基因相关矩阵（而不是 5000 × 5000），结果会非常奇怪但不一定报错——这是最危险的那种 bug。
>
> **务必用 `t()` 转置，并检查维度：行数应该等于样本数。**

## 选择软阈值（soft threshold）

WGCNA 中"加权"的含义是：两个基因之间的相关性不是简单的"有关/无关"（0 或 1），而是被提升到某个幂次（power）后变成一个连续的权重。这个幂次就是软阈值（soft threshold power）。

为什么要这么做？因为原始的相关性值（比如 r = 0.6）分布太平滑了，很难区分"真正的共表达"和"微弱的随机相关"。提升幂次后，强相关被保留、弱相关被大幅衰减，网络的"信噪比"提高了。

那 power 选多少？WGCNA 提供了一个工具来帮你选。

```r
# ============================================================
# Step 3: 选择软阈值 power
# pickSoftThreshold() 会尝试一系列 power 值，
# 计算网络在每个 power 下是否接近"无标度网络"（scale-free）
# 原则：选择使 scale-free 拓扑拟合指数 R² > 0.8 的
#        最小 power 值
# ============================================================

powers <- c(1:20)
sft <- pickSoftThreshold(
    datExpr,
    powerVector = powers,
    networkType = "signed",  # signed 网络区分正相关和负相关
    verbose = 0
)

# 找到 R² > 0.8 的最小 power
power_table <- sft$fitIndices
optimal_power <- power_table$Power[
    which(power_table$SFT.R.sq > 0.8)[1]
]

cat("软阈值选择结果：\n")
cat("  推荐 power:", optimal_power, "\n")
cat("  对应 R²:", round(power_table$SFT.R.sq[optimal_power], 3), "\n")
```

```
📊 输出：
软阈值选择结果：
  推荐 power: 5
  对应 R²: 0.806
```

💡 **为什么要追求"无标度网络"？**

> 真实的生物网络有一个特征：大多数基因只和少数其他基因有强关联，但少数"枢纽"基因和很多基因都有关联——这种"少数节点连接数极多、多数节点连接数极少"的分布叫做无标度（scale-free）分布。
>
> WGCNA 通过调整 power 来让构建的网络逼近这种分布。如果 power 太低，网络太"稠密"，噪声太多；如果 power 太高，网络太"稀疏"，信号也丢了。R² > 0.8 是一个经验阈值，表示网络已经足够接近无标度结构。

## 构建网络，识别模块

```r
# ============================================================
# Step 4: 一步法构建网络并识别模块
# blockwiseModules() 是 WGCNA 最核心的函数
# 它完成三件事：
#   1. 计算基因间的邻接矩阵（加权相关性）
#   2. 基于邻接矩阵计算 TOM（拓扑重叠矩阵）
#   3. 对 TOM 做层级聚类，动态剪切得到模块
# ============================================================

net <- blockwiseModules(
    datExpr,
    power = optimal_power,
    networkType = "signed",
    TOMType = "signed",
    minModuleSize = 30,    # 最小模块大小（少于 30 个基因的不成立）
    mergeCutHeight = 0.25, # 相似模块的合并阈值
    numericLabels = FALSE, # 用颜色命名模块（而不是数字）
    saveTOMs = FALSE,
    verbose = 0
)

# 查看模块信息
module_colors <- net$colors
module_table <- table(module_colors)

cat("模块识别结果：\n")
cat("  总模块数:", length(module_table), "\n")
cat("  各模块基因数:\n")
print(sort(module_table, decreasing = TRUE))
```

## 关键一步：模块和表型的关联

我们有了 7 个模块。但哪个模块和干旱胁迫最相关？

WGCNA 用"模块特征基因"（module eigengene, ME）来代表整个模块的表达模式——你可以把 ME 理解为该模块中所有基因表达量的"加权平均"。然后我们计算每个模块的 ME 和样本表型之间的相关性。

```r
# ============================================================
# Step 5: 模块-表型关联分析
# ============================================================

# 构建表型矩阵（将 "Control"/"Drought" 转为数值）
traits <- data.frame(
    Drought = as.numeric(sample_info$Group == "Drought")
)
rownames(traits) <- sample_info$Run

cat("表型矩阵（0=Control, 1=Drought）：\n")
print(traits)

# 计算模块特征基因
MEs <- net$MEs

# 计算模块-表型相关性
cor_result <- cor(MEs, traits, use = "p")
pval_result <- corPvalueStudent(cor_result, nSamples = nrow(datExpr))

cat("\n模块-表型相关性：\n")
module_trait <- data.frame(
    Module = gsub("ME", "", rownames(cor_result)),
    Correlation = round(as.numeric(cor_result), 3),
    P_value = signif(as.numeric(pval_result), 3)
) %>% arrange(P_value)

print(module_trait)
```

## 从关键模块中提取 hub gene

目标模块里并不是所有基因的地位都一样。有些基因和模块内其他基因的关联度特别高，它们是这个模块的"枢纽"（hub gene），很可能在调控网络中扮演核心角色。

WGCNA 用两个指标来衡量一个基因在模块中的重要性：

**模块成员度（Module Membership, MM）：** 该基因的表达与模块特征基因的相关性。MM 越高，说明该基因越"代表"这个模块的表达模式。

**基因显著性（Gene Significance, GS）：** 该基因的表达与感兴趣的表型（干旱处理）之间的相关性。GS 越高，说明该基因和干旱的直接关联越强。

**Hub gene = MM 高 + GS 高。** 既是模块核心、又与表型强相关的基因。

```r
# ============================================================
# Step 6: 提取与干旱最正相关模块的 hub gene
# ============================================================

# 选择与干旱最正相关的模块
target_module <- module_trait %>%
    filter(Module != "grey") %>%
    arrange(desc(Correlation), P_value) %>%
    slice(1) %>%
    pull(Module)

target_module_key <- paste0("ME", target_module)
cat("用于 hub gene 分析的目标模块:", target_module, "\n")

# 计算每个基因的 MM 和 GS
gene_MM <- cor(datExpr, MEs[[target_module_key]], use = "p")
gene_GS <- cor(datExpr, traits$Drought, use = "p")

# 提取目标模块中的基因
target_genes <- names(module_colors)[module_colors == target_module]

# 构建汇总表
hub_df <- data.frame(
    Gene = target_genes,
    MM = abs(gene_MM[target_genes, 1]),
    GS = abs(gene_GS[target_genes, 1])
) %>%
    arrange(desc(MM + GS))

cat(target_module, "模块 hub gene（MM + GS top 15）：\n")
print(head(hub_df, 15))
```

## WGCNA 能告诉你什么，不能告诉你什么

在结束之前，有一些重要的边界需要说清楚。

**WGCNA 能告诉你：**

- 哪些基因的表达模式高度协调（共表达模块）
- 哪个模块和你关心的表型最相关
- 模块中哪些基因处于核心位置（hub gene）
- 不同模块之间是否有关联（通过模块特征基因之间的相关性）

**WGCNA 不能告诉你：**

- **因果关系。** "基因 A 和基因 B 共表达"不等于"A 调控 B"或"B 调控 A"。它们可能都被同一个上游因子调控。
- **调控方向。** WGCNA 只看相关性，不看谁调控谁。如果你想推断调控关系，需要用 SCENIC 或者 decoupler 这样的工具——这是下一篇的内容。
- **功能验证。** Hub gene 是计算上的"核心"，但它是否在生物学上真正重要，需要实验验证（比如突变体表型分析）。

## 本篇小结

这一篇我们用 WGCNA 从"基因个体"视角切换到了"基因网络"视角。

核心发现：5,000 个高变异基因被分成了 38 个共表达模块。Blue 模块（983 个基因）和干旱处理高度正相关（r = 0.964），其功能富集在水分胁迫响应、转录调控和 ABA 响应。Turquoise 模块（1,280 个基因）高度负相关（r = -0.931）。Blue 模块的 hub gene 中，AT4G34000（ABF3）位于前列。

方法层面的收获：

**WGCNA 不是差异分析的替代品，而是补充。** 差异分析回答"哪个基因变了"，WGCNA 回答"哪些基因一起变"。两者的结合比单独使用任何一个都更有力。

**样本量很重要。** 我们的 6 个样本只能识别最强的模式。如果你的实验有 20 个以上样本（比如时间序列、多因素设计），WGCNA 的效果会好得多。

**Hub gene ≠ 因果基因。** 共表达不是因果关系。Hub gene 是进一步实验验证的候选列表，不是最终结论。
