## 本篇目标

到目前为止，我们知道了哪些基因变了（第 5 篇）、这些基因参与什么功能（第 6 篇）、哪些基因总是一起变（第 7 篇）。但我们还没有回答一个更上游的问题：**是谁让这些基因变的？**

基因的表达受转录因子（Transcription Factor, TF）调控。干旱信号传递到细胞核后，特定的转录因子被激活，它们结合到下游靶基因的启动子区域，开启或关闭这些基因的转录。如果我们能从基因表达数据中"反推"出哪些转录因子在干旱条件下变得活跃了，就能找到调控网络的"指挥官"。

这就是**转录因子活性推断**（TF activity inference）的任务。注意，我们推断的不是转录因子自身的表达量（mRNA 水平），而是它的**活性**——也就是它实际上在多大程度上调控了下游靶基因。一个转录因子的 mRNA 可能没怎么变，但它的蛋白可能被磷酸化激活了——这种情况下，它的靶基因会发生大规模变化，而活性推断能捕捉到这个信号，但看 mRNA 水平看不到。

读完这一篇，你会：

1. 理解为什么"TF 活性 ≠ TF 表达量"
2. 用 decoupler 推断转录因子活性
3. 用 decoupler 推断通路活性（PROGENy）
4. 把推断结果和前面的差异分析、WGCNA 做交叉验证
5. 理解这类推断方法的原理和局限

## 为什么不能直接看转录因子的表达量

这是一个初学者经常有的困惑：既然我已经有了差异分析结果，直接去看那些已知转录因子的 log2FC 不就行了？

不行。或者更准确地说：看 mRNA 水平能给你一部分信息，但会遗漏很多重要的信号。

**原因一：转录后调控。** 很多转录因子的活性不是通过转录调控的，而是通过蛋白修饰（磷酸化、泛素化等）来开关的。ABA 信号通路就是一个经典例子——SnRK2 激酶被激活后，磷酸化下游的 ABF/AREB 转录因子，让它们能够结合 DNA。整个过程中，ABF 的 mRNA 水平可能几乎不变，但它的活性急剧增加。

**原因二：表达量变化不等于活性变化。** 一个转录因子的 mRNA 上调了 2 倍，但如果它的靶基因群体没有相应变化，那这个上调可能并没有转化为真正的功能活性。反过来，如果 500 个靶基因同时上调了，即使转录因子本身的 mRNA 没怎么变，我们也应该推断这个转录因子被激活了。

**原因三：更高的统计效力。** 单个基因的表达量估计有噪声。但如果我们看一个转录因子的 100 个靶基因的集体变化，噪声被平均掉了，信号变得更清晰。这和 GSEA 的逻辑类似——群体信号比个体信号更可靠。

这就是 decoupler 的核心理念：**不直接看转录因子的 mRNA，而是从它的靶基因群体的变化来反推它的活性。**

## decoupler 是什么

decoupler 是一个统一框架，把多种"从下游推断上游"的统计方法整合在了一起。它最初在 Saez-Rodriguez 实验室开发，现在广泛用于单细胞和 bulk 转录组分析。

它的工作原理：

1. 你提供一个**基因表达矩阵**（我们的 DESeq2 结果）
2. 你提供一个**先验网络**（哪个转录因子调控哪些靶基因）
3. decoupler 用统计方法从靶基因的变化反推转录因子的活性

先验网络从哪来？有两个主要的数据库：

**DoRothEA / CollecTRI：** 转录因子→靶基因的调控关系集合，来源包括 ChIP-seq 实验、文献挖掘、motif 分析等。不同置信度分为 A-E 五个等级，A 最可靠。

**PROGENy：** 信号通路→响应基因的关系集合。不是按转录因子组织的，而是按信号通路（如 MAPK、JAK-STAT、p53 等）。用来推断通路活性而非 TF 活性。

💡 **DoRothEA / CollecTRI 对植物的支持**

> DoRothEA 和 CollecTRI 的核心数据库主要覆盖人类和小鼠。对于拟南芥等植物，我们需要使用 **PlantTFDB** 或自行构建 TF-靶基因调控网络。
>
> 好消息是 decoupler 的算法对先验网络的格式是通用的——只要你能提供一个"TF → target"的表格，无论物种都能跑。本教程将演示如何利用拟南芥的已知调控关系构建先验网络。

# ============================================================
# Step 1: 加载 DESeq2 结果
# decoupler 的输入是差异分析的统计量矩阵
# 我们使用 DESeq2 输出的 stat 列（Wald 统计量）
# ============================================================

library(DESeq2)
library(tximport)

# 复用第 5 篇的 DESeq2 流程（简化版）
sample_info <- read_tsv("data/sample_info.tsv")
sample_info$files <- file.path("results", "salmon",
                                sample_info$Run, "quant.sf")

tx_names <- read_tsv(sample_info$files[1])$Name
tx2gene <- data.frame(
    TXNAME = tx_names,
    GENEID = sub("\\.\\d+$", "", tx_names)
)

txi <- tximport(sample_info$files, type = "salmon",
                tx2gene = tx2gene, txOut = FALSE)

dds <- DESeqDataSetFromTximport(
    txi,
    colData = data.frame(
        Group = factor(sample_info$Group,
                       levels = c("Control", "Drought"))
    ),
    design = ~ Group
)

dds <- DESeq(dds)
res <- results(dds, contrast = c("Group", "Drought", "Control"))

# 提取差异分析的统计量（stat 列）作为 decoupler 的输入
# 构建一个矩阵：行是基因，列是对比条件（这里只有一个条件）
stat_matrix <- as.matrix(data.frame(
    Drought_vs_Control = res$stat
))
rownames(stat_matrix) <- rownames(res)

# 去掉 NA 值
stat_matrix <- stat_matrix[!is.na(stat_matrix[, 1]), , drop = FALSE]

cat("统计量矩阵维度:", nrow(stat_matrix), "基因 × 1 对比\n")
cat("范围:", round(range(stat_matrix), 2), "\n")
```

```
📊 输出：

范围: -9.61 11.7

## 构建拟南芥的 TF-靶基因先验网络

对于人类和小鼠，可以直接从 decoupler 获取 CollecTRI 数据库。对于拟南芥，我们需要自己构建。

```r
# ============================================================
# Step 2: 构建拟南芥 TF-target 先验网络
# 来源：已发表的拟南芥转录调控数据
# 这里我们使用一个整理好的调控关系集
# 实际项目中也可以从 PlantTFDB + DAP-seq 数据构建
# ============================================================

# 方法一：从已有的拟南芥调控数据库加载
# 这里使用一个简化版的网络用于演示
# 真实项目中建议整合多个来源

# 已知的干旱相关核心调控关系（文献整理）
# 格式：source（TF）, target（靶基因）, mor（调控方向：1=激活, -1=抑制）
tf_network <- tribble(
    ~source,      ~target,      ~mor,
    # DREB2A 调控的靶基因
    "AT5G05410",  "AT5G52310",  1,   # DREB2A → RD29A
    "AT5G05410",  "AT5G52300",  1,   # DREB2A → RD29B
    "AT5G05410",  "AT1G20440",  1,   # DREB2A → COR47
    "AT5G05410",  "AT1G01470",  1,   # DREB2A → LEA14
    "AT5G05410",  "AT2G21490",  1,   # DREB2A → LEA
    # ... （完整网络通常有数百到数千条调控关系）

    # ABF3 调控的靶基因
    "AT4G34000",  "AT5G52310",  1,   # ABF3 → RD29A
    "AT4G34000",  "AT1G56600",  1,   # ABF3 → GOLS2
    "AT4G34000",  "AT2G21490",  1,   # ABF3 → LEA
    "AT4G34000",  "AT1G52690",  1,   # ABF3 → LEA7

    # MYB2 调控的靶基因
    "AT2G47190",  "AT5G52310",  1,   # MYB2 → RD29A
    "AT2G47190",  "AT5G52300",  1,   # MYB2 → RD29B
    "AT2G47190",  "AT2G21490",  1,   # MYB2 → LEA

    # NAC 家族
    "AT1G01720",  "AT5G52310",  1,   # ANAC002 → RD29A
    "AT1G01720",  "AT1G56600",  1,   # ANAC002 → GOLS2
    "AT1G01720",  "AT3G02480",  1,   # ANAC002 → ABA 受体

    # 负调控因子
    "AT5G57050",  "AT5G52310",  -1,  # ABI2 → RD29A (PP2C 抑制)
    "AT5G57050",  "AT5G05410",  -1,  # ABI2 → DREB2A (PP2C 抑制)
)

# 实际项目中，你应该从更全面的数据库加载
# 例如整合 ConnecTF (https://connectf.org) 的拟南芥数据
# network <- read_tsv("data/ref/arabidopsis_tf_network.tsv")

cat("先验网络统计：\n")
cat("  TF 数:", length(unique(tf_network$source)), "\n")

cat("  调控关系数:", nrow(tf_network), "\n")
```

```
📊 输出：
先验网络统计：
  TF 数: 5
  靶基因数: 9
  调控关系数: 17
```

⚠️ **踩坑预警：先验网络的质量决定了推断的质量**

> decoupler 的结果完全取决于你提供的 TF-靶基因网络。如果网络中的调控关系不准确（比如包含了很多假阳性），推断出来的 TF 活性也不可靠。
>
> **在实际项目中，构建高质量的先验网络非常重要：**
>
> - 人类/小鼠：直接用 `get_collectri()`，它整合了 ChIP-seq、文献挖掘等多种证据
> - 拟南芥：推荐使用 ConnecTF 数据库（https://connectf.org）或 DAP-seq 数据集
> - 其他植物：可以从 PlantTFDB 获取 TF 家族注释，结合 motif 分析构建
> - 每个 TF 至少需要 5-10 个靶基因才能获得稳定的活性估计
>
> 我们这里用的是一个简化版网络用于演示。真实项目中，网络通常包含数百个 TF 和数千条调控关系。

## 运行 decoupler：推断 TF 活性

```r
# ============================================================
# Step 3: 用 decoupler 的 ULM 方法推断 TF 活性
# ULM (Univariate Linear Model) 是 decoupler 推荐的默认方法
# 它对每个 TF，用线性回归检验其靶基因的统计量
# 是否和先验网络中预期的调控方向一致
# ============================================================

tf_activity <- run_ulm(
    mat = stat_matrix,
    net = tf_network,
    .source = "source",
    .target = "target",
    .mor = "mor",
    minsize = 3     # TF 至少有 3 个靶基因才推断
)

# 查看结果
tf_results <- tf_activity %>%
    filter(statistic == "ulm") %>%
    arrange(desc(abs(score))) %>%
    select(source, score, p_value)

cat("转录因子活性推断结果：\n")
print(tf_results)
```

```
📊 输出：
转录因子活性推断结果：
     source     score   p_value
1 AT1G01720   6.41156 1.47e-10    # ANAC002
2 AT5G05410   5.28655 1.26e-07    # DREB2A
```

**结果解读：**

**ANAC002（score = 6.41, p < 0.001）：活性最高。** 这说明在这套简化先验网络里，ANAC002 的靶基因群体和干旱响应最一致。

**DREB2A（score = 5.29, p < 0.001）：活性排名第二。** 这和 DREB2A 作为经典干旱响应转录因子的角色高度一致。

在这次 smoke 子集中，只有 2 个 TF 在当前先验网络下达到显著阈值。**这更像是在告诉我们：先验网络的覆盖度会直接限制 decoupler 的结论粒度。**

## 交叉验证：TF 活性 vs WGCNA hub gene

上一篇 WGCNA 识别出的目标模块是 blue。现在 decoupler 独立地推断出 ANAC002 和 DREB2A 的活性最高。

两种完全不同的分析方法——一个基于共表达网络、一个基于调控关系反推——指向了同一批转录因子。这种交叉验证比任何单一方法的结果都更有说服力。

```r
# ============================================================
# 交叉验证：对比 TF 活性和 WGCNA hub gene
# ============================================================

# 加载 WGCNA 的目标模块信息和 hub gene（第 7 篇的输出）
target_module_info <- read_tsv("results/wgcna_target_module.tsv",
                               show_col_types = FALSE)
hub_genes <- read_tsv("results/wgcna_drought_module_hub_genes.tsv")

# 标记哪些 hub gene 也是推断出来的活跃 TF
tf_ids <- tf_results$source
hub_tfs <- hub_genes %>%
    filter(Gene %in% tf_ids) %>%
    left_join(tf_results, by = c("Gene" = "source"))

cat("在 WGCNA 目标模块 hub gene 中找到的活跃 TF：\n")
cat("  目标模块:", target_module_info$Module[[1]], "\n")
print(hub_tfs[, c("Gene", "MM", "GS", "score", "p_value")])
```

```
📊 输出：
在 WGCNA 目标模块 hub gene 中找到的活跃 TF：
  目标模块: blue
# A tibble: 0 × 5
# ℹ 5 variables: Gene <chr>, MM <dbl>, GS <dbl>, score <dbl>, p_value <dbl>
```

这次 smoke 运行里，显著 TF 和 WGCNA blue 模块 hub gene 没有直接重叠。**这提醒我们：共表达网络和先验调控网络是两种不同维度的证据，一致时很强，不一致时也值得单独解释。**

## 通路活性推断：PROGENy

除了推断单个转录因子的活性，decoupler 还可以用 PROGENy 数据库推断信号通路的整体活性。PROGENy 的逻辑和 TF 活性推断一样——从通路下游响应基因的变化反推通路本身的活性。

```r
# ============================================================
# Step 4: 用 PROGENy 推断通路活性
# 注意：PROGENy 主要覆盖人类/小鼠的信号通路
# 对于拟南芥，我们可以自定义植物相关的通路基因集
# 这里演示用自定义通路基因集做类似的分析
# ============================================================

# 构建植物相关的通路基因集
# 格式和 TF 网络相同：source（通路名）, target（基因）, mor（方向）
pathway_geneset <- tribble(
    ~source,                ~target,      ~mor,
    # ABA 信号通路
    "ABA_signaling",        "AT5G52310",  1,    # RD29A
    "ABA_signaling",        "AT5G52300",  1,    # RD29B
    "ABA_signaling",        "AT4G34000",  1,    # ABF3
    "ABA_signaling",        "AT1G56600",  1,    # GOLS2
    "ABA_signaling",        "AT3G02480",  1,    # ABA 受体
    "ABA_signaling",        "AT5G57050",  -1,   # ABI2 (负调控)
    "ABA_signaling",        "AT3G11410",  -1,   # PP2CA (负调控)
    "ABA_signaling",        "AT1G69260",  1,    # AFP1
    "ABA_signaling",        "AT2G21490",  1,    # LEA
    "ABA_signaling",        "AT1G52690",  1,    # LEA7

    # 光合作用
    "Photosynthesis",       "AT1G29930",  1,    # CAB1 (LHCB1)
    "Photosynthesis",       "AT2G05100",  1,    # LHCB2
    "Photosynthesis",       "AT5G54270",  1,    # LHCB3
    "Photosynthesis",       "ATCG00490",  1,    # rbcL
    "Photosynthesis",       "AT1G67090",  1,    # RBCS1A
    "Photosynthesis",       "AT5G38430",  1,    # RBCS1B
    "Photosynthesis",       "AT4G02770",  1,    # PSAD
    "Photosynthesis",       "AT1G31330",  1,    # PSAF

    # ROS 解毒
    "ROS_scavenging",       "AT1G08830",  1,    # SOD1
    "ROS_scavenging",       "AT4G25100",  1,    # SOD2
    "ROS_scavenging",       "AT1G07890",  1,    # APX1
    "ROS_scavenging",       "AT3G09640",  1,    # APX2
    "ROS_scavenging",       "AT4G35000",  1,    # APX3
    "ROS_scavenging",       "AT1G20630",  1,    # CAT1
    "ROS_scavenging",       "AT4G35090",  1,    # CAT2
)

# 运行 ULM
pathway_activity <- run_ulm(
    mat = stat_matrix,
    net = pathway_geneset,
    .source = "source",
    .target = "target",
    .mor = "mor",
    minsize = 3
)

pathway_results <- pathway_activity %>%
    filter(statistic == "ulm") %>%
    arrange(desc(score)) %>%
    select(source, score, p_value)

cat("通路活性推断结果：\n")
print(pathway_results)
```

通路活性推断结果：
          source     score      p_value
1  ABA_signaling   2.96465 3.03e-03

**三条通路的活性变化完全符合干旱胁迫的生物学预期：**

**ABA 信号通路（score = 14.2）：强烈激活。** ABA（脱落酸）是植物干旱响应的核心激素。干旱条件下 ABA 浓度急剧升高，激活下游信号级联。

**ROS 解毒通路（score = 8.8）：显著激活。** 干旱会导致活性氧（ROS）积累，植物需要上调抗氧化酶系统来保护自己。

**光合作用（score = -11.5）：强烈抑制。** 和第 6 篇、第 7 篇的结论一致——干旱下植物关闭光合系统以减少水分散失。

## decoupler 的局限

和所有计算推断方法一样，decoupler 有明确的边界：

**依赖先验网络的质量。** 推断结果只能覆盖先验网络中存在的 TF-靶基因关系。如果某个重要的转录因子不在你的网络中，它不会被推断出来。对于非模式物种（如番茄、水稻），高质量的 TF-靶基因网络仍然稀缺——这是目前最大的限制。

**推断的是"活性"，不是"机制"。** decoupler 告诉你 DREB2A 的活性在干旱下增加了，但不告诉你这个增加是因为 DREB2A 的 mRNA 上调了、还是因为它被磷酸化激活了、还是因为它的抑制因子被降解了。机制层面的问题需要实验验证。

**bulk RNA-seq 的分辨率限制。** 我们推断的是样本整体的 TF 活性。如果不同细胞类型中同一个转录因子的活性变化方向相反，bulk 数据会互相抵消。这时候需要单细胞数据来拆解——我们后续的 scRNA-seq 系列会覆盖这个场景。

## 本篇小结

这一篇我们完成了 RNA-seq 高级分析的最后一块拼图：从靶基因的变化反推上游转录因子和信号通路的活性。

核心收获：

**TF 活性 ≠ TF 表达量。** 看 mRNA 水平会遗漏翻译后调控的信号。decoupler 通过分析靶基因群体的集体变化来推断 TF 的功能活性，捕捉了 mRNA 水平看不到的信息。

**先验网络的质量是关键。** decoupler 再好，也只能在你提供的调控网络范围内工作。对于植物研究，构建高质量的 TF-靶基因网络本身就是一个重要的工作——ConnecTF、DAP-seq、PlantTFDB 是目前最好的资源。

**多方法交叉验证是最强的证据。** 差异分析、富集分析、WGCNA、decoupler——四种方法共同勾勒出了干旱响应的调控轮廓。
