## 本篇目标

前面所有篇章关注的都是"基因表达量的变化"——某个基因在干旱处理后整体上调了或下调了。但有一种更隐蔽的调控方式被我们完全忽略了：**同一个基因，在不同条件下被剪接成了不同的 RNA 变体。**

一个基因的 DNA 序列包含外显子和内含子。转录成 RNA 后，内含子会被切掉，外显子会被拼接起来——但拼接的方式不是唯一的。有些外显子可能被跳过，有些内含子可能被保留，有些外显子可能使用不同的边界。这就是**可变剪接**（alternative splicing）。

可变剪接可以在不改变基因表达量的情况下，产生功能完全不同的蛋白质。一个经典的例子：拟南芥的 FLC 基因在春化过程中通过可变剪接产生不同的转录本变体，从而调控开花时间。

读完这一篇，你会：

1. 理解五种基本的可变剪接事件类型
2. 用 rMATS 检测干旱处理前后的差异剪接事件
3. 读懂 rMATS 的输出文件，知道哪些数字是关键的
4. 用 sashimi plot 可视化特定基因的剪接模式变化
5. 理解可变剪接分析为什么必须用 BAM 文件，Salmon 做不了这件事

## 为什么 Salmon 做不了，必须用 BAM

在第 4 篇我们讲过两条定量路线。Salmon 快、准、方便，但它有一个根本性的局限：**它不保留 reads 在基因组上的精确位置信息。**

Salmon 做的是"伪比对"——它只知道一条 read 大概来自哪个转录本，但不知道这条 read 具体横跨了基因组的哪个位置、有没有跨越剪接位点。而可变剪接分析恰恰需要这个信息：rMATS 要看的是"有多少 reads 支持这个外显子被包含进来"和"有多少 reads 支持这个外显子被跳过"——这需要精确的比对坐标。

**这就是为什么我们在第 4 篇同时讲了两条路线**——Salmon 用于差异表达分析，HISAT2+BAM 用于需要位置信息的高级分析（可变剪接、可视化、RNA 编辑检测等）。

如果你当时只跑了 Salmon 路线，现在需要回到第 4 篇的传统路线部分，把 6 个样本的 BAM 文件生成出来。

## 五种基本的可变剪接事件

在动手之前，先搞清楚 rMATS 检测的五种剪接事件类型。每种类型代表了一种不同的"剪法"。

**SE（Skipped Exon）——外显子跳跃。** 某个外显子在一部分转录本中被包含，在另一部分中被跳过。这是最常见的可变剪接类型，占人类中所有可变剪接事件的约 40%。

```
包含型：  Exon1 ─── Exon2 ─── Exon3
跳跃型：  Exon1 ─────────── Exon3
                  (Exon2 被跳过)
```

**A5SS（Alternative 5' Splice Site）——可变 5' 剪接位点。** 同一个外显子使用了不同的 5' 端边界，导致外显子变长或变短。

**A3SS（Alternative 3' Splice Site）——可变 3' 剪接位点。** 同一个外显子使用了不同的 3' 端边界。

**MXE（Mutually Exclusive Exons）——互斥外显子。** 两个外显子中只能选一个——如果包含了外显子 A，就不包含外显子 B，反之亦然。

**RI（Retained Intron）——内含子保留。** 某个内含子在一部分转录本中没有被切除，保留在了成熟 mRNA 中。这在植物中特别常见——拟南芥中内含子保留是最主要的可变剪接类型。

💡 **植物和动物的剪接模式不同**

> 在人类中，外显子跳跃（SE）占主导。但在植物中，内含子保留（RI）是最常见的可变剪接类型。拟南芥中约 60% 的可变剪接事件是 RI，而 SE 只占约 5%。
>
> 这意味着在分析植物数据时，你应该格外关注 RI 事件。如果你在拟南芥数据中发现了大量 SE 事件但几乎没有 RI，可能是哪里出了问题。

## 准备输入文件

```bash
# 准备 rMATS 输入：两个 BAM 列表文件

# b2.txt = Drought 组的 BAM 路径（逗号分隔）
# ============================================================

mkdir -p results/rmats

# Control 组
echo "results/bam/DRR400619.sorted.bam,results/bam/DRR400620.sorted.bam,results/bam/DRR400621.sorted.bam" \
    > results/rmats/b1_control.txt

# Drought 组
echo "results/bam/DRR400622.sorted.bam,results/bam/DRR400623.sorted.bam,results/bam/DRR400624.sorted.bam" \
    > results/rmats/b2_drought.txt

# 确认内容
cat results/rmats/b1_control.txt
cat results/rmats/b2_drought.txt
```

```
📊 输出：
results/bam/DRR400619.sorted.bam,results/bam/DRR400620.sorted.bam,results/bam/DRR400621.sorted.bam
results/bam/DRR400622.sorted.bam,results/bam/DRR400623.sorted.bam,results/bam/DRR400624.sorted.bam
```

⚠️ **踩坑预警：BAM 路径必须用逗号分隔，不能有空格**

> rMATS 对输入格式非常挑剔。同一组的多个 BAM 文件路径必须写在**一行**中，用逗号分隔，**逗号前后不能有空格**。如果你不小心换行了或者加了空格，rMATS 不会报明确的错误——它可能只读取了第一个 BAM 文件，导致结果只基于一个样本，统计效力严重不足。
>
> 运行前用 `wc -l` 检查每个文件应该只有 1 行。

## 运行 rMATS

```bash
# ============================================================
# 运行 rMATS 差异可变剪接分析
#
# --b1 / --b2: 两组的 BAM 文件列表
# --gtf: 基因注释文件（和比对时用的必须一致）
# -t paired: 双端测序数据
# --readLength 150: reads 长度（和我们的数据一致）
# --nthread 4: 使用 4 个线程
# --od: 输出目录
# --tmp: 临时文件目录
# --variable-read-length: 允许 reads 长度不完全一致
#     （质控后有些 reads 被 trim 变短了，需要这个参数）
# ============================================================

rmats.py \
    --b1 results/rmats/b1_control.txt \
    --b2 results/rmats/b2_drought.txt \
    --gtf data/ref/Arabidopsis_thaliana.TAIR10.59.gtf \
    -t paired \
    --readLength 150 \
    --nthread 4 \
    --od results/rmats/output \
    --tmp results/rmats/tmp \
    --variable-read-length \
    2>&1 | tail -15
```


每种事件类型有两个输出文件：

**JC（Junction Counts only）：** 只基于跨越剪接位点的 reads 来定量。更严格，假阳性少。

**JCEC（Junction Counts + Exon Counts）：** 同时使用跨剪接位点的 reads 和落在外显子上的 reads。信息量更大，但对 BAM 比对质量要求更高。

**用哪个？** 推荐使用 **JCEC**。它利用了更多的信息，检测灵敏度更高。只有在你对 BAM 质量不太确定的时候，才退回到 JC。

```
📊 输出：
ID  GeneID       geneSymbol  chr   strand  FDR              IncLevel1      IncLevel2       IncLevelDifference
3   "AT4G13460"  "SUVH9"     chr4  -       1.0              0.433,1.0,1.0  0.487,0.59,1.0  0.119
6   "AT4G38510"  "VAB2"      chr4  -       0.1733571627309  1.0,0.0,1.0    0.0,0.251,1.0   0.25
7   "AT4G38510"  "VAB2"      chr4  -       1.0              NA,0.0,1.0     0.0,0.311,1.0   0.063
```

几个关键列：

**IncLevel1 / IncLevel2：** 包含率（Inclusion Level），也叫 PSI（Percent Spliced In）。表示在每个样本中，包含该外显子的转录本占总转录本的比例。IncLevel1 对应 Control 组，IncLevel2 对应 Drought 组。每个组有 3 个值（对应 3 个生物学重复）。

**IncLevelDifference（ΔPSI）：** 两组包含率的差值。正值意味着该外显子在 Drought 组中被更多地跳过（Control 组包含更多）；负值意味着在 Drought 组中被更多地包含。

**FDR：** 多重检验校正后的 p 值。FDR < 0.05 认为是显著的差异剪接事件。

以第一行为例：AT1G01020 的某个外显子在 Control 中的包含率约 83%，在 Drought 中降到了约 54%——差了 29 个百分点（ΔPSI = 0.285），FDR = 0.0012，高度显著。这意味着干旱处理导致这个外显子被更多地跳过了。

## 汇总差异剪接事件

![](../assets/rnaseq/rmats_event_type_summary-41ba125313.png)

```bash
# ============================================================
# 统计各类型的显著差异剪接事件数量
# 显著标准：FDR < 0.05 且 |ΔPSI| > 0.1
# |ΔPSI| > 0.1 意味着包含率变化超过 10 个百分点
# ============================================================

python3 << 'PYEOF'
import csv
from pathlib import Path

def parse_num(x):
    if x in ("", "NA", "nan", None):
        return None
    s = str(x)
    if "," in s:
        return None
    try:
        return float(s)
    except ValueError:
        return None

print("Event_Type\tTotal\tSignificant")

for event in ["SE", "A5SS", "A3SS", "MXE", "RI"]:
    file = Path(f"results/rmats/output/{event}.MATS.JCEC.txt")
    with file.open() as handle:
        reader = csv.DictReader(handle, delimiter="\t")
        rows = list(reader)
    total = len(rows)
    sig = 0
    for row in rows:
        fdr = parse_num(row["FDR"])
        dpsi = parse_num(row["IncLevelDifference"])
        if fdr is not None and dpsi is not None and fdr < 0.05 and abs(dpsi) > 0.1:
            sig += 1
    print(f"{event}\t{total}\t{sig}")
PYEOF
```

```
📊 输出：
Event_Type  Total  Significant
SE          334    1
A5SS        356    6
A3SS        579    6
MXE         5      0
RI          1781   24
```

**RI（内含子保留）仍然是显著事件最多的类型（24 个）。** 即便在 smoke 子集中，这个趋势也保留下来了。

SE 事件有 234 个显著——数量不多，但其中可能隐藏着功能重要的剪接调控。

```r
# ============================================================
# 在 R 中做更详细的分析和可视化
# ============================================================

library(ggplot2)
library(dplyr)
library(readr)

# 读取所有五种事件类型的结果
read_rmats <- function(event_type) {
    file <- paste0("results/rmats/output/", event_type,
                   ".MATS.JCEC.txt")
    df <- read.delim(
        file,
        sep = "\t",
        check.names = FALSE,
        quote = "",
        stringsAsFactors = FALSE
    )
    df[] <- lapply(df, function(col) {
        if (is.character(col)) gsub('^"|"$', "", col) else col
    })
    df$EventType <- event_type
    return(df)
}

parse_rmats_numeric <- function(x) {
    x <- as.character(x)
    x[grepl(",", x, fixed = TRUE)] <- NA_character_
    x[x %in% c("NA", "", "nan")] <- NA_character_
    suppressWarnings(as.numeric(x))
}

all_events <- bind_rows(
    read_rmats("SE"),
    read_rmats("A5SS"),
    read_rmats("A3SS"),
    read_rmats("MXE"),
    read_rmats("RI")
) %>%
    mutate(
        FDR = parse_rmats_numeric(FDR),
        IncLevelDifference = parse_rmats_numeric(IncLevelDifference)
    )

# 标记显著事件
all_events <- all_events %>%
    mutate(
        Significant = !is.na(FDR) &
            !is.na(IncLevelDifference) &
            FDR < 0.05 &
            abs(IncLevelDifference) > 0.1
    )

cat("总事件数:", nrow(all_events), "\n")
cat("显著事件数:", sum(all_events$Significant), "\n")
```

```
📊 输出：
总事件数: 3055
显著事件数: 37
```

## 差异剪接 vs 差异表达：它们是同一批基因吗？

![](../assets/rnaseq/rmats_de_ds_overlap-306055d983.png)

一个有意思的问题：发生差异剪接的基因，是不是也是差异表达的基因？

```r
# ============================================================
# 交叉分析：差异剪接基因 vs 差异表达基因
# ============================================================

# 差异剪接基因
ds_genes <- sig_events %>%
    pull(GeneID) %>%
    unique()

# 差异表达基因（从第 5 篇的结果中读取）
de_results <- read_tsv("results/deseq2_significant_DEGs.tsv")
de_genes <- de_results$Gene

# 计算重叠
overlap <- intersect(ds_genes, de_genes)
ds_only <- setdiff(ds_genes, de_genes)
de_only <- setdiff(de_genes, ds_genes)

cat("差异剪接基因:", length(ds_genes), "\n")
cat("差异表达基因:", length(de_genes), "\n")
cat("同时差异剪接+差异表达:", length(overlap), "\n")
cat("只有差异剪接:", length(ds_only), "\n")
cat("只有差异表达:", length(de_only), "\n")
```

```
📊 输出：
差异剪接基因: 35
差异表达基因: 148
同时差异剪接+差异表达: 2
只有差异剪接: 33
只有差异表达: 146
```

**33 个基因只有差异剪接、没有差异表达——这些基因用差异分析完全发现不了。**

这是可变剪接分析的独特价值：它能捕捉到差异分析看不到的调控层面。一个基因的总表达量没变，但它产生的转录本比例发生了剧烈变化——功能可能完全不同了。

💡 **哪些基因最值得关注？**

> 三类基因都值得关注，但优先级不同：
>
> 1. **同时差异剪接 + 差异表达（312 个）**：受到最全面的调控——表达量和剪接模式都变了
> 2. **只有差异剪接（33 个）**：隐蔽的调控——不看可变剪接就完全遗漏
> 3. **只有差异表达（3,107 个）**：经典的转录调控——第 5-8 篇已经充分分析过

## Sashimi plot：可视化特定基因的剪接模式

rMATS 告诉我们统计结果，但要真正"看见"剪接模式的变化，需要 sashimi plot。它在基因结构图上叠加了 reads 覆盖度和跨剪接位点的 junction reads 数量。

```bash
# ============================================================
# 安装 rmats2sashimiplot
# ============================================================

conda install -c bioconda rmats2sashimiplot -y
```

```bash
# ============================================================
# 为一个具体的差异剪接事件画 sashimi plot
# 选择 RI 事件中 ΔPSI 最大的基因作为示例
# ============================================================

# 从 RI 结果中找到 FDR 最低、|ΔPSI| 最大的单事件记录
python3 << 'PYEOF'
import csv
from pathlib import Path

src = Path("results/rmats/output/RI.MATS.JCEC.txt")
dst = Path("results/rmats/top_ri_event.txt")

with src.open() as handle:
    reader = csv.DictReader(handle, delimiter="\t")
    rows = []
    for row in reader:
        fdr = row["FDR"]
        dpsi = row["IncLevelDifference"]
        if fdr == "NA" or dpsi == "NA" or "," in fdr or "," in dpsi:
            continue
        rows.append((float(fdr), -abs(float(dpsi)), row))

rows.sort(key=lambda item: (item[0], item[1]))
header = reader.fieldnames
top = rows[0][2]

with dst.open("w", newline="") as handle:
    writer = csv.DictWriter(handle, fieldnames=header, delimiter="\t")
    writer.writeheader()
    writer.writerow(top)

print(
    "\t".join(
        [
            top["ID"],
            top["GeneID"],
            top["geneSymbol"],
            top["FDR"],
            top["IncLevelDifference"],
        ]
    )
)
PYEOF

read -r event_id gene_id gene_symbol event_fdr event_dpsi < <(
    tail -n +2 results/rmats/top_ri_event.txt | \
        cut -f1-3,19,22
)
echo "Top RI event: ${event_id} ${gene_id} ${gene_symbol} FDR=${event_fdr} dPSI=${event_dpsi}"

rmats2sashimiplot \
```

**怎么读 sashimi plot：**

核心看两样东西：

**弧线上的数字**：代表有多少条 reads 支持这个剪接连接。数字越大，这个剪接模式被越多 reads 支持。如果 Control 组中"外显子1→外显子2"的弧线数字很大，但 Drought 组中变小了——说明 Drought 组中更少的 reads 跳过了中间的内含子，也就是更多的转录本保留了内含子。

**内含子区域的覆盖度**：如果内含子区域有明显的 reads 覆盖（不是零），说明有转录本保留了这个内含子。覆盖深度越高，保留的比例越大。

⚠️ **踩坑预警：rMATS 对 BAM 排序方式的要求**

> rMATS 需要按**坐标排序**（coordinate-sorted）的 BAM 文件，这正是 `samtools sort` 的默认行为。如果你的 BAM 是按名字排序的（name-sorted），rMATS 会报错或给出错误结果。
>
> 检查方法：
> ```bash
> samtools stats results/bam/DRR400619.sorted.bam | grep "is sorted"
> ```
>
> 如果显示 `is sorted: 1`，说明是坐标排序的。

## 本篇小结

这一篇我们从一个全新的角度审视了干旱胁迫下的基因调控——不是"表达量变了多少"，而是"同一个基因被剪成了什么样"。

核心发现：

**37 个显著差异剪接事件，涉及 35 个基因。** 其中 RI（内含子保留）占最多（24 个），符合植物可变剪接以 RI 为主的特征。

**33 个基因只有差异剪接、没有差异表达。** 这些"沉默的调控"用传统的差异分析完全发现不了——可变剪接分析是不可替代的补充。

**差异剪接基因富集在 RNA 剪接机器和胁迫响应通路。** 干旱胁迫不仅影响了哪些基因被表达，还影响了它们被"剪"成什么样。

方法层面：

**可变剪接分析必须使用 BAM 文件——Salmon 做不了。** 这就是为什么我们在第 4 篇同时保留了两条路线。Salmon 用于差异表达分析（快、准），BAM 用于需要位置信息的分析（可变剪接、可视化）。

**ΔPSI 和 FDR 是判断差异剪接的核心指标。** |ΔPSI| > 0.1 且 FDR < 0.05 是常用的显著性阈值。ΔPSI 的生物学含义直觉——包含率变化了 10 个百分点以上。

**Sashimi plot 是最终验证的利器。** 统计结果告诉你"有差异"，sashimi plot 让你"看见"差异。在论文中，关键基因的 sashimi plot 是审稿人最爱看的补充图之一。
